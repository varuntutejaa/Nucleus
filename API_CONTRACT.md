# Nucleus API Contract

Base URL (local dev): `http://127.0.0.1:8000`
Interactive docs (auto-generated by FastAPI): `GET /docs`

All responses are JSON. There is no auth, no database, and no write endpoints
— this is a read-only, in-memory demo service (see project non-goals).

---

## `GET /api/alerts/raw`

Returns the full raw alert stream, sorted ascending by time.

### Query parameters

| Param    | Type                    | Default       | Notes                                                                 |
|----------|-------------------------|---------------|------------------------------------------------------------------------|
| `source` | `"synthetic" \| "dataset"` | `"synthetic"` | `synthetic` = generator described in README; `dataset` = real AIOps2020-derived alerts (bundled ~675-alert sample, or regenerated live from a local raw dataset copy if `NUCLEUS_AIOPS_DATASET_ROOT` is set) |
| `as_of`  | float (unix seconds)    | none          | If set, only alerts with `timestamp_unix <= as_of` are returned. Used by the frontend's replay mode to simulate alerts arriving live. |

### Response

```json
{
  "alerts": [
    {
      "id": "alt-00001",
      "timestamp": "2026-07-15T15:17:26.409209+00:00",
      "timestamp_unix": 1784128646.4092093,
      "service": "payment-db",
      "severity": "warning",
      "severity_rank": 2,
      "message": "payment-db: TLS certificate renewal reminder (25 days remaining)",
      "source": "synthetic"
    }
  ],
  "count": 800
}
```

- `severity` is one of `critical | warning | info`; `severity_rank` is `3 | 2 | 1` respectively (higher = more severe), included so the frontend/root-cause tiebreaker don't need a lookup table.
- `source` on each alert reflects where *that alert* actually came from. If `source=dataset` is requested but the dataset fails to load/parse, the service transparently falls back to the synthetic generator — every alert in the response will honestly say `"source": "synthetic"` in that case rather than lying about its origin.
- Ground-truth incident labels used internally by the synthetic generator (for the sanity-check script and our own tuning) are never included in this response.

## `GET /api/alerts/correlated`

Runs the full correlation pipeline (composite distance + sliding-window HDBSCAN + cross-window merge + root-cause selection) and returns clusters, standalone noise, and metrics.

### Query parameters

| Param    | Type   | Default | Range    | Notes |
|----------|--------|---------|----------|-------|
| `source` | `"synthetic" \| "dataset"` | `"synthetic"` | — | same as above |
| `alpha`  | float  | `0.25`  | `0.0–1.0` | semantic-distance weight |
| `beta`   | float  | `0.45`  | `0.0–1.0` | temporal-distance weight |
| `gamma`  | float  | `0.30`  | `0.0–1.0` | service-topology-penalty weight |
| `as_of`  | float (unix seconds) | none | — | same replay cutoff as `/raw` |

`alpha`/`beta`/`gamma` do not need to sum to 1 — they're renormalized server-side so relative weighting is always well-defined (see `app/pipeline/distance.py`).

### Response

```json
{
  "clusters": [
    {
      "cluster_id": "cluster-1",
      "root_cause": {
        "id": "alt-00006",
        "timestamp": "2026-07-15T15:29:57.204913+00:00",
        "timestamp_unix": 1784129397.2049127,
        "service": "payment-service",
        "severity": "critical",
        "severity_rank": 3,
        "message": "payment-service: CPU utilization sustained at 94% for 5m",
        "source": "synthetic"
      },
      "suppressed": [
        { "...": "same alert shape as root_cause" }
      ],
      "size": 59,
      "time_span_seconds": 315.4,
      "explanation": "Earliest alert in cluster (critical severity) among 59 correlated alerts on 2 service(s)."
    }
  ],
  "noise": [
    { "...": "same alert shape, alerts HDBSCAN could not confidently cluster" }
  ],
  "metrics": {
    "raw_count": 800,
    "cluster_count": 13,
    "noise_count": 186,
    "suppressed_count": 601,
    "reduction_pct": 75.1,
    "embedding_backend": "sentence-transformers/all-MiniLM-L6-v2",
    "weights_used": { "alpha": 0.25, "beta": 0.45, "gamma": 0.3 }
  }
}
```

- `cluster_id` is assigned after sorting clusters chronologically by root-cause timestamp (`cluster-1` is the earliest incident).
- `size == 1 + len(suppressed)` always (root cause + its suppressed alerts).
- `explanation` is a one-sentence, human-readable statement of the root-cause heuristic actually applied to that cluster (earliest timestamp, severity tiebreaker), per the project's explainability requirement — not a generic string.
- `noise` holds every alert HDBSCAN did not place in any cluster (across all sliding windows). These are **never dropped** — they're returned in full, same shape as any other alert.
- `metrics.reduction_pct = 100 * (1 - (cluster_count + noise_count) / raw_count)`, i.e. the percentage of raw alerts suppressed into a cluster's root cause rather than shown standalone in the primary view.
- `metrics.embedding_backend` reports which embedding path actually served this request: `sentence-transformers/all-MiniLM-L6-v2` normally, or `tfidf-fallback` if the sentence-transformers model could not be loaded (see README's reliability notes).
- `metrics.weights_used` echoes the *renormalized* alpha/beta/gamma actually applied.

---

## Errors

Both endpoints return standard FastAPI validation errors (HTTP 422) if `alpha`/`beta`/`gamma` are outside `[0, 1]`, and never throw on a bad/missing dataset file — see "Demo reliability" in README.md.
